GoogleGo 代码规范

命名约定

Go 一般不用_

  • 文件名全小写,_分隔单词 例:my_file.go
  • 包名称 package 全小写 例:tabwriter
    • 避免被局部变量覆盖 例:usercount > count
    • *_test 用于测试的包才有 _
  • struct 名每个单词大写开头 例:ListNode
  • 变量名驼峰式小写单词开头 例:myBook
  • 函数名驼峰式 例:createTable
  • 函数参数命名 短,类型本身的缩写 例:func(ri *ResearchInfo)
  • 常量一般每个单词首字母大写
    • 导出常量(别的 package 可以调用的)以大写字母开头 例:MaxSize=10
    • 未导出的常量以小写字母开头 例:minSize=10
  • 函数和方法名称不应使用 Get 或 get 前缀
    • 一个函数返回两个或多个相同类型的参数,添加名称会很有用. 例:func (n *Node) Children() (left, right *Node, err error)
  • 注:导出到包外的话首字母都要大写

注释

  • 包注释必须出现在包内语句的上方
  • 避免与其他导入的名称冲突使用重命名导入 例:import (
    fspb “path/to/package/foo_service_go_proto”
    )
  • TODO 注释(临时的,短期的解决方案,或已经够好但仍不完美的代码使用 TODO 注释)
  • etc
